数据结构 —— 顺序表

什么是顺序表

顺序表是线性表的一种存储方式。它的底层就是数组——在内存中申请一段连续的空间，依次存放数据元素。

数组本身是静态的（长度固定），而顺序表是在数组外面包了一层，实现了动态管理：可以随时添加元素、删除元素、自动扩容。

顺序表和链表的核心区别：

  • 顺序表：内存连续，支持随机访问（O(1) 访问任意位置）
  • 链表：内存离散，只能顺序访问，但插入删除更灵活

二、创建与释放

2.1 创建顺序表

为什么表头和数据空间要分两次 malloc？

因为它们是两块独立的内存：

  • 表头（SeqList 结构体）存的是”管理信息”——指向数据区的指针、长度、容量
  • 数据空间（data）才是真正存元素的地方

如果合在一起，扩容的时候就没办法只扩大数据区了。

2.2 释放顺序表

释放顺序和创建顺序相反：先释放数据区，再释放表头。每次 free 之后顺手置 NULL，是好习惯。

三、插入操作

3.1 尾插法（pushback）

尾插法效率最高——**时间复杂度 O(1)**，因为不需要移动任何元素，直接在末尾追加即可。

3.2 任意位置插入（insert）

核心操作：从后往前移动元素

为什么必须从后往前？——因为 data[i+1] = data[i]，如果用从前往后移动，前面的元素会覆盖后面的元素，造成数据丢失。

**时间复杂度 O(n)**：最坏情况是在头部插入，所有元素都要后移。

四、删除操作

核心操作：从前往后移动元素

插入是从后往前，删除是从前往后——因为 data[i] = data[i+1]，用后面的元素覆盖前面的。

**时间复杂度 O(n)**：最坏情况是删除头部元素，后面所有元素都要前移。

注意这里用的是”按值删除”——先找到值对应的位置，再删除。也可以用”按位置删除”（直接传 pos），更常见的做法是把 find 和 delete 分开。

五、查找操作

六、扩容机制

关于 static 关键字：这里把 enlargerSeqList 声明为 static，意思是只能在本文件内调用。对外部来说，扩容是内部实现细节，不需要暴露给用户。这是 C 语言实现封装的一种手段。

为什么是 2 倍扩容？

  • 如果一次只扩 1 个空间，每次插入都要扩容，效率极低
  • 2 倍扩容是一种常见的折中策略——既不会频繁扩容，又不会浪费太多内存
  • STL 中 vector 也是类似的思路（不同编译器倍数不同，通常是 1.5~2 倍）

八、复习要点总结

操作 时间复杂度 关键要点
创建 O(1) 分两次 malloc：表头 + 数据区
尾插 O(1) 直接放 data[length] 即可
任意位置插入 O(n) 从后往前移动元素
删除 O(n) 从前往后移动元素
查找 O(n) 顺序遍历
扩容 O(n) 2 倍扩容 + 拷贝数据

面试常问的几个细节：

  1. 为什么长度和容量要分开记录？ — 容量是总空间，长度是已用空间，扩容时靠容量判断
  2. 插入时为什么 pos 可以等于 length？ — 等于 length 就相当于尾插
  3. 删除后需不需要缩容？ — 一般不需要，这就是”以空间换时间”的思想
