二、预处理（Preprocessing）

把人类写的代码变成”纯机器能读的代码”

预处理是四步中最不被人注意的一步——因为大多数时候它都在幕后自动完成。但它做的事情非常重要。

2.1 预处理干了什么

预处理器的任务可以概括为四个字：文本替换。具体包括：

操作 原始代码 预处理后
头文件展开 #include <stdio.h> 把 stdio.h 的全部内容复制进来
宏替换 #define MAX 100
int arr[MAX];		 int arr[100];
条件编译 #ifdef DEBUG
printf("debug\n");
#endif		 如果定义了 DEBUG 就保留，否则删掉
删除注释 // 这是一行注释 注释被空白替代

2.2 为什么需要预处理

预处理解决的核心问题是代码复用。

你写的程序需要用 printf，但你的 .c 文件里并没有 printf 的代码——它只是在 stdio.h 里声明了一下。预处理阶段，预处理器会把 stdio.h（以及里面层层包含的所有头文件）全部在你当前文件里展开。展开后，你的源文件从几十行变成了几千行——里面有了 printf 的声明，编译器于是知道”有这么一个函数，参数是这样的，返回值是这样的”。

四、汇编（Assembly）

把汇编代码翻译成机器码

4.1 汇编器在做什么

汇编这一步相对简单——把人类可读的汇编助记符（mov, push, call 等）翻译成机器可以直接执行的机器码（0 和 1 组成的二进制指令）。

汇编器的输入是 .s 汇编文件，输出是 .o 目标文件。

4.2 目标文件里有什么

**这里藏着最重要的概念——“未解析的符号”**。

你的 main.o 里调用了 printf，但 printf 的实现不在 main.o 里——它在系统的 C 库（libc）里。所以 main.o 的符号表里，对 printf 标记的是”未定义的，需要别人提供”。

同样，如果你写了一个多文件项目：

它们各自编译成 main.o 和 add.o：

  • main.o 的符号表里：add 是”未定义的，需要别人提供”
  • add.o 的符号表里：add 是”已定义的，提供给人用”

“undefined reference” 的本质就是：在链接时，有个符号在所有 .o 文件里都没找到定义。比如你忘了编译 add.c，那 add.o 根本不存在，链接器找不到 add，就报 undefined reference to 'add'。

回到最初的问题——为什么 F5 调试报 “undefined reference”？

VS Code 自动生成的 tasks.json 里，编译参数写的是 ${file}——只编译当前打开的那个文件。所以当你按 F5 调试 main.c 时，它等价于执行了：

add.c 没有被编译！add.o 不存在！链接器找不到 add 的定义——啪，undefined reference to 'add'。

修法很简单：把 ${file} 改成 ${fileDirname}\\*.c，让编译器把当前目录里所有 .c 文件都编译进同一个可执行文件。

六、编译命令速查表

目标 GCC 命令 说明
一步到可执行文件 gcc main.c add.c -o main 四步自动走完
只预处理 gcc -E main.c -o main.i 看到头文件展开后的样子
只编译到汇编 gcc -S main.c -o main.s 看到汇编代码
只编译+汇编 gcc -c main.c -o main.o 生成 .o 目标文件
只看符号表 nm main.o 看”定义了啥””需要啥”
查看目标文件结构 objdump -h main.o 看到代码段、数据段

七、静态库与动态库

你的资料里有两个文件夹——static/ 和 dynamic/。这就是库的两种形式。

7.1 静态库（.a / .lib）

本质：一堆 .o 文件打成的包。

链接时，链接器把库里的代码拷贝一份到你的可执行文件里。

优点 缺点
运行时不需要库文件 每个程序都有一份拷贝，占空间
不依赖外部环境 库更新后需要重新编译链接
部署简单 可执行文件体积大

7.2 动态库（.so / .dll）

本质：库代码留在.so文件里，可执行文件里只记录”我需要这个库”。

优点 缺点
多个程序共享一份库 部署时需要带上或确保安装了库文件
库升级不需要重编译程序 运行时查找库失败程序就崩了
可执行文件体积小 不同版本的库可能不兼容

7.3 工作中的类比

静态库 (.a) 动态库 (.so / .dll)
类比 文章里把别人的文字全文引用进来 文章里给出参考文献，让读者自己去查
链接时机 编译时（生成 exe 的时候） 运行时（双击 exe 之后）
体积 exe 变大 exe 不变
更新 库改了，重新编译 exe 库改了，换一个 .so 文件就行

十、核心概念对照表

概念 一句话解释
预处理 展开 #include、替换 #define、删除注释
编译 把 C/C++ 翻译成汇编语言
汇编 把汇编翻译成机器码（.o 文件）
链接 把多个 .o + 库拼成一个可执行文件
.o 目标文件 编译和汇编的输出，包含机器码 + 符号表
符号表 记录”我提供了什么”和”我需要什么”
undefined reference 链接时发现需要的符号没有任何地方提供定义
静态库 .a 链接时把库代码拷进 exe
动态库 .so 运行时才加载库代码
CMake 帮你生成 Makefile 的构建工具
