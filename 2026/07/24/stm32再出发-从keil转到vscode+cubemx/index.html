STM32再出发——从Keil转到VSCode+CubeMX

发表于 2026-07-24 分类于 嵌入式
本文字数： 5.2k 阅读时长 ≈ 5 分钟
从零开始，搭建 VS Code + STM32CubeMX 的嵌入式开发工具链，生成第一段代码，编译烧录，点亮第一盏灯。记录整个过程中遇到的所有坑和解决方法。

前言

大一上的时候，我曾短暂学习过 STM32，当时用的是 Keil + 寄存器 的开发方式。Keil 的界面老旧、代码补全基本没有、所有内容全靠硬写，或者说硬抄。由于当时面临很多压力，所以我暂时终止了STM32的学习。

这次重新出发，我决定换一套更现代化的工具链：

  • STM32CubeMX —— 图形化配置引脚和外设，生成初始化代码
  • VS Code + STM32 扩展 —— 现代化的代码编辑器，补全、跳转、Git 集成
  • HAL 库 —— 用库函数操作外设，先建立工程能力，以后再深入底层

最终效果：通过 VS Code 管理项目，只用写核心的功能代码，一键编译，一键烧录到开发板，LED 灯点亮并闪烁。

一、工具链概览

先搞清楚每样东西是干什么的：

工具 作用
STM32CubeMX 图形化配置芯片引脚、时钟、外设，生成初始化 C 代码
VS Code 代码编辑器，用于编写业务逻辑
STM32 VS Code Extension ST 官方插件，集成编译、烧录、调试功能
GCC Toolchain 编译器（arm-none-eabi-gcc），把代码编译成机器码
ST-Link 调试器/烧录器，通过 SWD 接口连接电脑和开发板

整个工作流程：

二、安装步骤

2.1 安装 STM32CubeMX

  1. 去 ST 官网 下载 STM32CubeMX（需要注册 ST 账号，免费）
  2. 运行安装程序，一直下一步
  3. 安装过程中它会自动下载芯片支持包（STM32Cube FW_F1 等），等它下载完就行

安装完成后桌面上会有个 STM32CubeMX 的图标。

2.2 安装 VS Code 与 STM32 扩展

  1. 去 VS Code 官网 下载安装
  2. 安装完成后，打开 VS Code，点左侧的扩展图标（或按 Ctrl+Shift+X）
  3. 搜索 **”STM32”**，找到 STMicroelectronics 官方出的 **”STM32 VS Code Extension”**，点击安装
  4. 安装完成后，VS Code 会自动检测并建议安装一些配套工具，确认即可

这个扩展帮我们集成了编译和烧录的功能，不用自己手动配 CMake 和命令行。

2.3 安装 ST-Link 驱动

ST-Link 是连接电脑和开发板的调试器。插上电脑后，如果设备管理器里显示黄色感叹号，说明没装驱动。

解决方法：

  1. 下载 ST-Link 驱动（en.stsw-link009.zip）
  2. 解压后，64 位系统运行 dpinst_amd64.exe
  3. 安装完成后，重新插拔 ST-Link
  4. 打开设备管理器，应该能看到 STMicroelectronics STM32 STLink，没有黄色感叹号

如果不想单独装驱动，安装 STM32CubeProgrammer 也会自动装好 ST-Link 驱动。

三、创建第一个项目

3.1 在 CubeMX 中新建项目

  1. 打开 STM32CubeMX
  2. 点击 “New Project”
  3. 在搜索框输入 STM32F103C8T6（我用的是 Keysking大佬开发的集成开发板），双击选中
  4. 弹窗问是否初始化外设，点 “No”

3.2 配置引脚

在 CubeMX 的 Pinout View视图中，芯片的每个引脚都可以点击配置。

配置 LED 灯引脚（PA7）：

  1. 在芯片图上找到 PA7，点击它

  2. 选择 GPIO_Output

  3. 在左侧的 Categories面板下面的System Core里找到 GPIO 设置，点击后中间会跳出Configuration界面，如下图所示

  5. 选中PA7引脚，下方跳出具体的设置界面

  6. 此处本来GPIO output level是low，也就是低电平，把它改成High,高电平即可

  8. 其他保持默认：推挽输出、无上下拉、低速度

3.3 配置调试接口

切换到 Pinout & Configuration → 搜索 SYS：

  • Debug 选择 Serial Wire
  • 这样 PA13/SWDIO 和 PA14/SWCLK 会被自动配置，用于 ST-Link 调试

这一步很重要！否则烧录一次后，下次 SWD 就被禁用了，再也烧不进新程序。

3.4 选择工具链

切换到 Project Manager 标签页：

设置项 值
Project Name stm32code-demo
Project Location 选一个纯英文目录–此处我选的是D盘下的stm32project目录
Toolchain / IDE CMake（关键！一定要选 CMake，而不是其他的）
Default Compiler/Linker GCC

点击右上角 **”Generate Code”**，生成代码。弹窗问要不要打开项目，选 **”Open Project”**，它会自动用 VS Code 打开。

4.4 关于 USER CODE 区块

这是 CubeMX 最贴心的设计之一。每次 CubeMX 重新生成代码时：

  • ✅ USER CODE BEGIN/END 之间的内容会被保留
  • ❌ 外部的代码会被完全覆盖

所以养成习惯：所有自己写的代码，都放在 USER CODE 区块里。

常用的 USER CODE 位置：

区块 位置 用途
USER CODE BEGIN 2 MX_GPIO_Init() 之后 初始化设置
USER CODE BEGIN WHILE while(1) 内部开头 主循环逻辑
USER CODE BEGIN 3 while(1) 内部结尾 主循环末尾
USER CODE BEGIN 0 main.c 顶部函数区 自定义函数

五、编译

写好了代码，怎么编译成开发板能运行的机器码？

5.1 通过 VS Code 编译

在 VS Code 底部状态栏，ST 扩展提供了按钮：

  1. 先点 齿轮图标 “生成” → 开始编译
  2. 观察底部终端面板的输出
  3. 看到类似下面的输出就是编译成功了：

其中 RAM 和 FLASH 显示了固件占用的空间，生成完毕 和 退出代码为 0 代表编译成功。

编译产物在 build/Debug/ 目录下：

  • stm32code-demo.elf — 可执行文件（包含调试信息）
  • stm32code-demo.bin — 纯二进制文件（用于烧录）

5.2 常见编译错误

错误 原因 解决
undefined reference to XXX_Pin 引脚宏被删了但代码还在引用 去 CubeMX 重新配引脚并生成代码
arm-none-eabi-gcc: not found 工具链没装上 重新安装 STM32 VS Code Extension
CMake Error: toolchain file not found 工具链路径不对 确保 STM32 扩展已正确安装

六、烧录

编译通过后，把程序下载到开发板上。

6.1 硬件连接

ST-Link 和开发板通过 SWD 接口连接，只需要 4 根线：

ST-Link 开发板
SWDIO PA13
SWCLK PA14
GND GND
3.3V VCC

有些 ST-Link 还多一根 NRST（复位）线，也可以接上，但不是必须的。

6.2 驱动问题排坑

插上 ST-Link 后，如果 VS Code 检测不到设备，大概率是驱动没装。

排坑步骤：

  1. 打开设备管理器（Win + X → 设备管理器）
  2. 看有没有 STM32 STLink 出现
  3. 如果有黄色感叹号 → 运行 dpinst_amd64.exe 安装驱动
  4. 装好驱动后，重新插拔 ST-Link
  5. 设备管理器里应该显示正常

6.3 烧录到开发板

驱动正常后，在 VS Code 中：

  1. 点击左侧的 运行和调试 图标（🦖 或 Ctrl+Shift+D），打开运行和调试面板
  2. 确认下方 STM32Cube设备与开发板 列表中能看到 STM32 STLink（![屏幕截图 2026-07-24 204737](./../images/stm32再出发-从keil转到vscode+cubemx/屏幕截图 2026-07-24 204737.png)
  3. 点击上方的 运行和调试 按钮，选择 STM32Cube:STM32 Launch STLink GDB Server![屏幕截图 2026-07-24 205138](./../images/stm32再出发-从keil转到vscode+cubemx/屏幕截图 2026-07-24 205138.png)
  4. 此时 ST-Link 开始连接开发板，终端会输出连接日志
  5. 点击 VS Code 上方调试工具栏中的 继续 按钮（▶️ 图标）![屏幕截图 2026-07-24 205322](./../images/stm32再出发-从keil转到vscode+cubemx/屏幕截图 2026-07-24 205322.png)
  6. 程序开始运行，LED 灯闪烁 🎉

6.4 如何确认烧录成功

烧录完成后，如果代码写的是闪烁逻辑，会看到：

  • LED 以 500ms 的间隔一亮一灭 💡
  • 这说明：程序成功烧录 → 芯片运行 → GPIO 正常翻转

如果灯不亮：

现象 可能原因
完全不亮 接线反了？引脚号对不对？初始电平设的是 High 吗？
一直亮不闪 while(1) 里没写 TogglePin 或 Delay
亮一下就不亮了 可能芯片复位了，检查电源

七、总结

7.1 与传统 Keil 开发的对比

对比项 Keil 方案 VS Code + CubeMX 方案
界面 老旧，2000 年代风格 现代化，美观
代码编辑 无补全 智能补全、跳转定义
配置方式 手动写寄存器或配图形界面 CubeMX 统一图形化配置
项目迁移 难跨平台 基于 CMake，跨平台
费用 收费（商业版） 全部免费
学习曲线 陡峭（要学很多很多很多细节） 平缓（HAL 封装好了，只关心核心功能）

7.3 后续学习路线

点灯是嵌入式开发的 **”Hello World”**，下一步可以学：

  • 外部中断 —— 按键控制 LED
  • 定时器/PWM —— 呼吸灯、舵机控制
  • UART 串口 —— 和电脑通信，打印调试信息
  • ADC —— 读取传感器数值
  • I2C/SPI —— 驱动 OLED 屏幕、传感器模块

每条路线跑通后，我都会写博客记录下来，欢迎继续关注 😄
