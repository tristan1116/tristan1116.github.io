二、SSH 概念与原理

2.1 什么是 SSH？

SSH（Secure Shell）是一种加密的网络协议，用于在不安全的网络中安全地进行远程登录、命令执行和文件传输。

特性 说明
默认端口 22（TCP）
加密方式 对称加密 + 非对称加密 + 哈希
认证方式 密码认证 / 公钥认证（推荐）
协议版本 SSH-2（当前标准）

2.2 SSH 的应用场景

场景 工具/方式 说明
远程登录 ssh user@host 从本地安全登录到远程服务器
文件传输 SCP / rsync / SFTP 加密传输文件
端口转发 ssh -L / ssh -R 建立安全隧道
远程执行 ssh user@host 'command' 无需登录执行单条命令
Git 认证 SSH Key 免密推送到 GitHub 等平台

五、SSH 安全最佳实践

序号 措施 配置 安全等级
1 禁用 root 登录 PermitRootLogin no ⭐⭐⭐⭐⭐
2 使用密钥认证 PubkeyAuthentication yes ⭐⭐⭐⭐⭐
3 关闭密码认证（密钥配好后） PasswordAuthentication no ⭐⭐⭐⭐
4 更换默认端口 Port 2222（减少自动扫描） ⭐⭐⭐
5 限制登录用户 AllowUsers dev admin ⭐⭐⭐⭐
6 配置防火墙 sudo ufw allow 22/tcp ⭐⭐⭐⭐
7 保护私钥 chmod 600 ~/.ssh/id_ed25519 ⭐⭐⭐⭐⭐
8 限制认证尝试次数 MaxAuthTries 3 ⭐⭐⭐
9 设置空闲超时 ClientAliveInterval 300 ⭐⭐⭐
10 使用 fail2ban 自动封禁暴力破解 IP ⭐⭐⭐⭐

六、常见问题与解决

问题 原因 解决方法
ssh: command not found 未安装 SSH 客户端 sudo apt install openssh-client -y
Connection refused SSH 服务未启动或防火墙阻止 sudo systemctl start ssh；检查 sudo ufw status
Permission denied (publickey) 公钥未正确配置 检查远程 ~/.ssh/authorized_keys（权限600）和 ~/.ssh（权限700）
Bad permissions 错误 ~/.ssh 或密钥文件权限过宽 chmod 700 ~/.ssh；chmod 600 ~/.ssh/id_*
免密登录仍然要求密码 PubkeyAuthentication 未启用 检查远程 /etc/ssh/sshd_config 中 PubkeyAuthentication yes
rsync: command not found 未安装 rsync sudo apt install rsync -y
Host key verification failed 远程主机密钥变更 ssh-keygen -R <hostname> 删除旧密钥后重新连接
WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED 主机密钥不匹配 确认是否为正常变更后执行 ssh-keygen -R <IP>

七、实验总结

通过本次实验，系统掌握了 SSH 远程访问的完整技能栈：

  1. SSH 服务管理： 学会了 OpenSSH 的安装、配置和安全加固，理解了 sshd_config 中各项配置的含义
  2. 密钥认证： 掌握了 Ed25519 密钥对的生成、分发和免密登录的实现原理，推荐将密码认证关闭以提升安全性
  3. 连接优化： 使用 ~/.ssh/config 文件实现了连接别名，大幅简化了多服务器管理中的 SSH 操作
  4. 文件传输： 掌握了 SCP（简单直接）和 rsync（增量同步、功能更强）两种远程传输工具，能根据实际场景选择合适的方案
  5. 实际应用： 将 rsync 应用到了 Hexo → Nginx 博客部署的实际场景中，打通了本地开发到远程部署的完整链路

SSH 是 Linux 运维的核心技能，本次实验掌握的知识将直接应用于实验8（CI/CD 自动化部署）的远程部署环节。
