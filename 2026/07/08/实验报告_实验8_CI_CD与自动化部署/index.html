实验目标： 掌握 CI/CD 的概念，使用 GitHub Actions 实现 Hexo 博客的自动化构建和部署

一、实验目的

  1. 理解 CI/CD（持续集成/持续交付/持续部署）的核心概念和价值
  2. 掌握 GitHub Actions Workflow 的编写方法
  3. 实现 Hexo 博客的自动构建和部署（方案A：GitHub Pages）
  4. 掌握 Nginx 静态文件服务器的配置（方案B：自有服务器）
  5. 学会使用 rsync 进行增量文件同步部署
  6. 理解 CI/CD 工具链中各环节的职责和协作方式

二、CI/CD 概念

2.1 什么是 CI/CD？

缩写 全称 含义
CI Continuous Integration 持续集成 — 代码变更后自动构建、测试，尽早发现集成问题
CD Continuous Delivery 持续交付 — 自动将构建产物发布到可部署状态（需手动触发部署）
CD Continuous Deployment 持续部署 — 自动将构建产物直接部署到生产环境（全自动）

2.2 CI/CD 的价值

  • 自动化 — 消除手动构建和部署的人为错误
  • 快速反馈 — 代码推送后几分钟内看到部署结果
  • 一致性 — 每次构建使用相同的环境，消除”在我机器上能跑”问题
  • 可追溯 — 每次部署都有记录，方便回滚和审计
  • 提升效率 — 开发者专注于编写代码，而非手动部署操作

四、实验结果

成功标准

  • .github/workflows/deploy.yml Workflow 文件创建成功
  • Personal Access Token 创建并配置为 GitHub Secret PERSONAL_TOKEN
  • Push 代码后 GitHub Actions 自动触发并所有步骤执行成功
  • GitHub Pages 网站内容自动更新
  • Nginx 站点配置文件 nginx/blog-server-block 创建完成
  • rsync 部署脚本 nginx/deploy.sh 创建完成
  • 可选：Nginx 服务器上博客正常访问

两种方案对比

维度 GitHub Actions + Pages Nginx + rsync
部署触发 全自动（git push） 手动执行脚本
服务器需求 不需要（GitHub 托管） 需要自有 Ubuntu Server
成本 免费（公开仓库） 服务器费用
HTTPS 自动提供 Let’s Encrypt 需手动配置证书
访问速度 全球 CDN 加速 取决于服务器带宽
灵活性 受限于 GitHub Pages 能力 完全可控（支持任何配置）
学习价值 CI/CD 工作流实践 服务器运维实践
适用场景 个人博客、文档站 企业内部站点、定制化部署

五、常见问题与解决

问题 原因 解决方法
Actions 报 PERSONAL_TOKEN 认证失败 Secret 名称不一致或 Token 过期 确认 Secret 名称完全一致（大写），Token 有 repo 权限且未过期
部署后 404 页面 GitHub Pages 未正确配置 检查仓库 Settings → Pages 中 Branch 和文件夹设置
部署后样式丢失/资源 404 _config.yml 中 url 或 root 不正确 确保 url: https://<username>.github.io 和 root: /
Actions 一直处于 Queued 状态 所用仓库为私有仓库且达到 Actions 免费额度 升级 GitHub 计划或改为公开仓库
npm install 步骤很慢 无缓存或网络问题 Workflow 中已配置 actions/cache@v3，第二次构建会显著加速
Nginx 返回 403 Forbidden 文件权限不正确 sudo chown -R $USER:$USER /var/www/blog
Nginx 测试配置报错 语法错误 sudo nginx -t 查看具体错误行号
rsync 推送失败 SSH 无法连接远程服务器 确认 SSH 服务运行、密钥已配置、防火墙已放行

六、实验总结

通过本次实验，将课程所学的多个工具串联为一个完整的自动化部署流程：

  1. CI/CD 理解： 深入理解了持续集成和持续部署的概念，体会了自动化带来的效率提升和错误减少
  2. GitHub Actions： 掌握了 Workflow 的编写方法，包括触发条件、Job 配置、Step 编排、Secret 管理和缓存优化
  3. 两种部署方案：
    • 方案A（GitHub Actions + Pages）实现了 git push → 自动发布 的全自动流程
    • 方案B（Nginx + rsync）提供了自有服务器的部署能力，给了更大的灵活性和控制力
  4. 综合实践： 本实验是整个课程的综合实践，将前面的工具（Hexo、Git、Markdown、VS Code）和 Linux 运维知识（SSH、rsync、Nginx）串联为一个完整的自动化部署体系
  5. 安全实践： 学会了使用 GitHub Secrets 管理敏感信息（Token），以及 Nginx 安全头配置

本实验标志着《软件开发工具实践》课程 Web 主线的终点：从本地编写 Markdown 文章，到 Git 版本控制，到 Hexo 静态生成，到 GitHub Actions 自动构建，再到 GitHub Pages / Nginx 自动部署——实现了一条完整的现代 Web 开发与自动化部署流水线。
