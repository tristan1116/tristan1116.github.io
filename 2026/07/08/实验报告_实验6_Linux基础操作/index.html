实验目标： 掌握 Linux 命令行基本操作、用户管理、文件权限和环境变量配置

一、实验目的

  1. 掌握 Linux Shell 命令行的基本操作方法
  2. 理解 Linux 用户和组的管理机制
  3. 掌握文件权限的设置方法（数字模式和符号模式）
  4. 学会环境变量在会话级、用户级、系统级的配置方式
  5. 理解 Shell 配置文件加载顺序

二、Linux 命令行基础

2.1 命令提示符解读

部分 含义
dev 当前登录用户名
node1 主机名（hostname）
~ 当前工作目录（~ = 用户主目录 /home/dev）
$ 普通用户提示符（# 表示 root 超级用户）

2.2 Shell 常用快捷键

快捷键 功能 场景
Tab 自动补全命令/路径/文件名 输入命令时
Ctrl + C 终止当前正在运行的命令 卡住/死循环时
Ctrl + D 退出终端 / 发送 EOF 退出登录
Ctrl + L 清屏（等效 clear 命令） 清理屏幕
↑ / ↓ 浏览历史命令 重复执行
Ctrl + R 搜索历史命令 查找之前用过的命令
Ctrl + A / E 光标跳转到行首/行尾 编辑长命令

权限表示解读

数字权限对照

数字 二进制 权限 含义
7 111 rwx 读 + 写 + 执行
6 110 rw- 读 + 写
5 101 r-x 读 + 执行
4 100 r– 只读
3 011 -wx 写 + 执行
2 010 -w- 只写
1 001 –x 仅执行
0 000 — 无权限

常用权限组合：

数字 权限 适用场景
755 rwxr-xr-x 可执行脚本、目录
644 rw-r–r– 普通配置文件
600 rw——- 私密文件（SSH密钥）
777 rwxrwxrwx 临时共享目录（不安全，避免）

五、常见问题与解决

问题 原因 解决方法
sudo: command not found sudo 未安装 使用 root 安装：apt install sudo
Permission denied 权限不足 检查权限 ls -l，使用 sudo 提权或 chmod 修改
修改 ~/.bashrc 后未生效 未重新加载 执行 source ~/.bashrc 或重新登录
找不到新建用户 用户未实际创建 cat /etc/passwd 确认；检查 adduser 输出
rm -rf 误删文件 操作失误 删除前务必再三确认路径；考虑使用 trash-cli 替代
脚本提示 Permission denied 缺少执行权限 chmod +x script.sh 添加执行权限
export 的变量重启后丢失 仅在当前会话生效 写入 ~/.bashrc 或 /etc/environment 持久化

六、实验总结

通过本次实验，系统掌握了 Linux 操作系统的基础技能：

  1. 命令行操作： 熟练掌握了文件和目录的创建、查看、复制、移动、删除等基本操作，以及系统资源监控命令
  2. 用户管理： 理解了 /etc/passwd 文件的字段含义，掌握了用户的完整生命周期管理（创建 → 授权 → 切换 → 删除）
  3. 权限管理： 深入理解了 Linux 的 rwx 权限模型，能灵活运用数字模式（chmod 755）和符号模式（chmod u+x）设置权限
  4. 环境配置： 掌握了环境变量在会话级（export）、用户级（~/.bashrc）、系统级（/etc/profile.d/）三种级别的配置方法，理解了 Shell 配置文件加载顺序
  5. 效率提升： 学会了使用 alias 创建命令别名、配置历史记录、自定义提示符等实用技巧

本次实验掌握的 Linux 基础操作是后续实验7（SSH 远程访问）和实际运维工作的必备技能。
