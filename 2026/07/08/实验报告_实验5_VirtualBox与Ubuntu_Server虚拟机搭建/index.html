实验5：VirtualBox 与 Ubuntu Server 虚拟机搭建

实验日期： 2026年7月7日 - 7月8日
实验环境： Windows 11 + Oracle VirtualBox 7.x + Ubuntu Server 22.04 LTS
实验目标： 掌握 VirtualBox 虚拟机管理，搭建 Ubuntu Server 虚拟集群，理解虚拟化网络模式

一、实验目的

  1. 理解虚拟化技术的基本概念和应用场景
  2. 掌握 VirtualBox 的安装和虚拟机创建流程
  3. 理解 NAT、Bridged、Host-Only 三种网络模式的区别与适用场景
  4. 学会使用 Netplan 配置 Linux 网络（静态 IP、DNS）
  5. 掌握虚拟机克隆和批量部署的方法
  6. 为后续 Linux 实验搭建基础环境

二、虚拟化概念与网络模式

2.1 什么是虚拟化？

虚拟化技术允许在一台物理计算机上同时运行多个独立的操作系统实例（虚拟机，VM）。每个 VM 拥有独立的 CPU、内存、磁盘和网络资源，彼此隔离。

应用场景：

  • 开发环境隔离（不污染宿主机）
  • 多操作系统兼容性测试
  • 服务器集群模拟
  • 学习 Linux 操作系统

2.2 VirtualBox 网络模式详解

模式 IP 来源 访问外网 主机 ↔ VM VM 间通信 外部访问 VM 适用场景
NAT VirtualBox DHCP ✅ ❌（需端口转发） ❌ ❌ 仅需上网
Bridged 物理网络 DHCP ✅ ✅ ✅ ✅ 模拟真实网络
Host-Only VirtualBox DHCP ❌ ✅ ✅ ❌ 内部开发网络
NAT + Host-Only 混合 ✅ (NAT) ✅ (Host-Only) ✅ (Host-Only) ❌ 本实验采用

本实验采用 NAT + Host-Only 组合方案：

  • NAT 网卡（enp0s3）： 用于访问外网（安装软件包、更新系统）
  • Host-Only 网卡（enp0s8）： 用于宿主机与 VM 之间、VM 与 VM 之间的内部通信

三、实验步骤

步骤1：安装 VirtualBox

  1. 从 https://www.virtualbox.org/ 下载 Windows 版本
  2. 默认选项安装（确保勾选网络组件）
  3. 重要： 如果提示 “VT-x/AMD-V 不可用”，需重启进入 BIOS 开启虚拟化技术

步骤2：下载 Ubuntu Server ISO

从 https://ubuntu.com/download/server 下载 Ubuntu Server 22.04 LTS ISO 镜像文件。

步骤3：创建模板虚拟机

3.1 基本配置

参数 值 说明
名称 ubuntu-template 作为后续克隆的模板
类型 Linux 操作系统类型
版本 Ubuntu (64-bit) 64位架构
内存 2048 MB 2GB，满足服务器基本需求
虚拟硬盘 VDI，20 GB，动态分配 按需增长，节省物理磁盘

3.2 网络配置

在 VM 设置 → 网络：

  • 网卡1： 启用 → 连接方式：NAT（enp0s3）
  • 网卡2： 启用 → 连接方式：Host-Only Adapter（enp0s8）

步骤4：安装 Ubuntu Server

  1. 挂载 Ubuntu Server ISO → 启动 VM
  2. 选择 “Try or Install Ubuntu Server“
  3. 选择语言：English（推荐英文避免乱码）
  4. 网络配置：保持 DHCP 自动获取
  5. 存储配置：选择 “Use an entire disk“ → Done → Continue
  6. 关键步骤： 在 Software Selection 页面，使用空格键勾选 “Install OpenSSH server“
  7. 创建用户（例如：dev）和密码
  8. 等待安装完成 → Reboot Now

成功标准

  • VirtualBox 安装成功
  • Ubuntu Server 22.04 LTS 安装在模板 VM 上
  • OpenSSH server 在安装时勾选

五、常见问题与解决

问题 原因 解决方法
“VT-x/AMD-V 不可用” BIOS 中虚拟化未开启 重启进入 BIOS → 开启 Intel VT-x 或 AMD-V
Host-Only 网络不通 VirtualBox Host-Only 网络未正确配置 检查 VirtualBox 全局设置 → 网络 → Host-Only Networks
克隆后网络不通 MAC 地址冲突或网卡名称变化 重新初始化 MAC 地址；ip a 查看网卡名称是否匹配 Netplan 配置
NAT 无法访问外网 NAT 网卡未启用或配置错误 确保 enp0s3 配置为 dhcp4: yes
netplan apply 报错 YAML 缩进不正确 YAML 严格使用空格缩进（不能使用 Tab），检查对齐
主机名解析不生效 hosts 文件未更新 确认 /etc/hosts 中添加了所有节点的 IP-主机名映射
SSH 连接被拒绝 SSH 服务未安装或未启动 安装：sudo apt install openssh-server -y；启动：sudo systemctl start ssh

六、实验总结

通过本次实验，成功掌握了虚拟化技术的基础操作：

  1. 虚拟化理解： 深入理解了虚拟化的概念和价值，能够在单台物理机上运行多台隔离的 Linux 服务器
  2. 网络配置： 掌握了 NAT、Bridged、Host-Only 三种网络模式的区别，采用 NAT + Host-Only 组合方案实现了外网访问和内部通信的兼顾
  3. Netplan 配置： 学会了 Ubuntu Server 中使用 Netplan（YAML 格式）配置静态 IP 和 DNS
  4. 批量部署： 通过虚拟机克隆技术，从模板 VM 快速部署了 3 个配置一致的节点，大幅提升了效率
  5. 集群基础： 配置了 hosts 文件实现节点间主机名互解析，为后续实验做好了网络基础

本实验搭建的 4 节点虚拟集群（template + node1/2/3）将作为实验6（Linux 基础操作）和实验7（SSH 远程访问）的基础实验环境。
