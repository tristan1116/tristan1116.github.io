五、常见问题与解决

问题 原因 解决方法
'hexo' 不是内部命令 Hexo CLI 未安装或 PATH 问题 npm install -g hexo-cli，确保 Node.js 添加到 PATH
hexo init 失败 网络问题或 npm 源慢 配置镜像：npm config set registry https://registry.npmmirror.com
hexo server 端口冲突 4000 端口被占用 使用 hexo server -p 4001 切换端口
NexT 主题不显示 _config.yml 中 theme 未修改 确认 theme: next 配置正确
code 命令不可用 安装时未勾选 PATH VS Code内按 Ctrl+Shift+P → “Install ‘code’ command in PATH”
Markdown 预览不显示 Mermaid 未使用正确预览插件 使用 Markdown Preview Enhanced 预览
粘贴图片路径错误 Paste Image 配置问题 检查 pasteImage.path 和 basePath 设置
hexo deploy 报 403 Git 认证失败 使用 Personal Access Token 代替密码
GitHub Pages 显示 404 仓库名或分支不正确 确认仓库名为 <username>.github.io，Pages 分支选 main
部署后样式丢失 url 和 root 配置错误 确保 _config.yml 中 url 与 Pages 地址一致
ERROR Deployer not found 未安装部署插件 npm install hexo-deployer-git --save

六、实验总结

通过实验2-4，成功搭建了一套完整的 “编写 → 生成 → 部署” 个人博客工作流：

  1. 环境搭建（实验2）： 在 Windows 上安装配置了 Node.js、Git、Hexo CLI，成功创建标准 Hexo 项目并安装 NexT 主题，实现了本地博客预览
  2. 编辑工具链（实验3）： 配置了 VS Code 编辑器和 Markdown 写作环境，掌握了 Markdown 核心语法和 Mermaid 图表，显著提升了文档和博客文章的编写效率
  3. 部署发布（实验4）： 深入理解了 SSG 与 CMS 的架构差异，成功将 Hexo 博客部署到 GitHub Pages，实现了免费、安全的静态网站托管
  4. 技术收获： 掌握了 编写(Markdown) → 版本控制(Git) → 静态生成(Hexo) → 自动部署(GitHub Pages) 的现代 Web 开发基础工具链，为后续实验8的 CI/CD 自动化打下了基础

本阶段搭建的 Hexo 博客项目和 GitHub Pages 托管将作为实验8（CI/CD 自动化部署）的基础平台。
