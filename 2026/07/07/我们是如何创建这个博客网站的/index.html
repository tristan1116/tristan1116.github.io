实验二 Web软件开发环境

一、实验目标

  1. 安装并运行一个Web博客软件（本实验选用 Hexo 静态博客框架）；
  2. 了解和掌握Web开发环境的基础安装与配置（Node.js、npm、Git）；
  3. 理解Web软件的B/S架构特点以及静态站点与动态站点的区别。

二、背景知识

2.1 静态博客 vs 动态博客

类型 原理 特点 典型代表
静态博客 在本地将Markdown等源文件预编译为纯HTML页面 加载速度快、安全性高、无需数据库 Hexo、Hugo、VuePress、Jekyll
动态博客 每次用户访问时由服务器端程序动态生成页面 功能丰富、有后台管理界面、需要数据库 WordPress、Typecho、Halo

2.2 Web开发环境的核心组件

组件 作用 本实验对应的工具
运行时环境 执行编程语言代码 Node.js（运行JavaScript/Hexo）
包管理器 管理项目依赖库 npm（Node Package Manager）
版本控制 代码管理与协作 Git
构建工具 将源文件转换为可部署的网站 Hexo CLI
本地服务器 在开发时预览网站效果 Hexo自带的server模块

2.1 代码编辑器 vs 集成开发环境（IDE）

类型 特点 典型代表

2.2 插件（Extension）的概念

插件是VS Code最核心的扩展机制。通过安装不同的插件，可以为编辑器增加语法高亮、代码补全、格式化、预览、调试等新功能。

2.3 Markdown简介

Markdown是一种轻量级标记语言，通过简单的纯文本语法实现结构化文档的编写，并支持转换为HTML、PDF等格式。

Markdown的核心特点：

  • 简洁性：仅需少量符号（如#、*、-）即可完成排版
  • 可读性：未渲染的纯文本仍然清晰易读
  • 兼容性：支持GitHub、各大博客平台、笔记工具等场景

二、背景知识

2.1 什么是静态站点生成器（SSG）？

静态站点生成器（Static Site Generator, SSG）是一类工具，它将Markdown等源文件和模板文件作为输入，预先编译成完整的HTML、CSS、JavaScript静态网页。它的核心理念是“提前做好”，而非“现场制作”。

静态站点 vs 动态站点：

对比维度 静态站点（SSG） 动态站点（如WordPress）
页面生成时机 构建时预先生成所有页面 用户访问时由服务器实时生成
数据库 不需要 通常需要
访问速度 极快（纯静态文件） 较慢（需执行后端代码+查数据库）
安全性 高（无后端攻击面） 需额外防护
部署方式 上传静态文件即可 需配置服务器环境和数据库
适用场景 博客、文档站、个人网站 电商、社交、CMS

2.2 为什么选择Hexo？

Hexo是一个基于Node.js的快速、简洁且高效的博客框架。它使用Markdown解析文章，在几秒内即可利用靓丽的主题生成静态网页。Hexo拥有400+主题和丰富的插件生态，适合技术博客和个人网站。

二、背景知识

2.1 什么是虚拟化技术？

虚拟化技术允许在一台物理计算机（宿主机）上同时运行多个独立的操作系统（虚拟机），每个虚拟机都拥有自己独立的CPU、内存、硬盘和网络资源。

2.2 虚拟机网络的三种核心模式

网络模式 特点 典型用途
NAT 虚拟机通过宿主机上网，外部不可直达 日常上网、安装软件
桥接（Bridged） 虚拟机直接接入宿主机所在局域网 需要被其他设备直接访问
仅主机（Host-Only） 宿主机与虚拟机形成隔离私网 搭建内网互通实验环境

本实验将采用 “NAT + 仅主机”双网卡方案。

三、实验环境

  • 操作系统：Windows 10/11
  • 网络：需要互联网连接
  • 硬件要求：CPU支持VT-x/AMD-V虚拟化技术；内存≥8GB；硬盘剩余空间≥50GB
  • 所需软件：
    • Oracle VirtualBox
    • Ubuntu Server 22.04 LTS ISO镜像

二、背景知识

2.1 什么是SSH？

SSH（Secure Shell）是一种加密网络协议，用于在不安全的网络中安全地远程登录和执行命令。SSH默认使用22端口。

2.2 SSH的两种认证方式

认证方式 原理 安全性 适用场景
密码认证 每次登录输入密码 较低 临时访问
密钥认证 使用公钥/私钥对验证身份 高 日常管理、自动化脚本

2.3 文件传输工具

工具 特点 适用场景
SCP 基于SSH的简单复制 一次性、少量文件传输
rsync 增量同步，只传输变化的部分 目录同步、定期备份
