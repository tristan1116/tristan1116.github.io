题目描述

石头剪刀布是常见的猜拳游戏：石头胜剪刀，剪刀胜布，布胜石头。如果两个人出拳一样，则不分胜负。在《生活大爆炸》第二季第 8 集中出现了一种石头剪刀布的升级版游戏。

升级版游戏在传统的石头剪刀布游戏的基础上，增加了两个新手势：

斯波克:《星际迷航》主角之一。

蜥蜴人:《星际迷航》中的反面角色。

现在，小 A 和小 B 尝试玩这种升级版的猜拳游戏。已知他们的出拳都是有周期性规律的，但周期长度不一定相等。例如：如果小 A 以 石头-布-石头-剪刀-蜥蜴人-斯波克 长度为 的周期出拳,那么他的出拳序列就是 石头-布-石头-剪刀-蜥蜴人-斯波克-石头-布-石头-剪刀-蜥蜴人-斯波克-...，而如果小 B 以 剪刀-石头-布-斯波克-蜥蜴人 长度为 的周期出拳,那么他出拳的序列就是 剪刀-石头-布-斯波克-蜥蜴人-剪刀-石头-布-斯波克-蜥蜴人-...。

解题思路

破解本题的关键是把提供的猜拳表格翻译成二维数组胜负表，通过取模运算在周期数组中定位每次的出拳，再利用胜负表快速判断结果

1. 存储出拳规律

题目输入的第二行和第三行分别给出了小A和小B的出拳规律，我们用两个数组来存储：

  • **数组 a[]**：存储小A的出拳周期，长度为 na
    • 例如：a[0] = 0 表示小A第一次出”剪刀”
    • a[1] = 1 表示小A第二次出”石头”
    • 以此类推，小A的出拳会按照这个数组循环
  • **数组 b[]**：存储小B的出拳周期，长度为 nb
    • 例如：b[0] = 0 表示小B第一次出”剪刀”
    • b[1] = 3 表示小B第二次出”蜥蜴人”
    • 以此类推，小B的出拳会按照这个数组循环

数组与周期序列的对应关系：

  • 小A第 i 次出拳：a[i % na]
    • 当 i = 0 时，0 % 6 = 0，出 a[0]
    • 当 i = 6 时，6 % 6 = 0，又出 a[0]，完成一个周期循环
  • 同理，小B第 i 次出拳：b[i % nb]

2. 胜负表的构建

题目给出了5种手势的胜负关系，我们用 5×5 的二维数组 来存储所有可能的结果：

  • score[甲][乙] = 1 表示甲赢
  • score[甲][乙] = 0 表示平局
  • score[甲][乙] = -1 表示甲输（乙赢）

数组下标对应关系：

  • 0 → 剪刀
  • 1 → 石头
  • 2 → 布
  • 3 → 蜥蜴人
  • 4 → 斯波克

3. 得分统计

遍历 0 到 N-1 次猜拳：

  1. ia = i % na：计算小A本次出拳在数组 a[] 中的位置
  2. ib = i % nb：计算小B本次出拳在数组 b[] 中的位置
  3. a[ia] 和 b[ib] 就是双方本次的手势
  4. 查表 score[a[ia]][b[ib]] 得到结果
  5. 根据结果累加对应玩家的得分

复杂度分析

  • 时间复杂度：O(N)，只需要遍历一次猜拳过程
  • 空间复杂度：O(NA + NB)，存储两个周期数组

注意事项

  1. 数组 a[] 和 b[] 存储的是出拳规律，不是出拳序列本身
  2. 通过 i % na 和 i % nb 的取模运算实现周期循环
  3. 胜负表中 score[a[ia]][b[ib]] 的查表方式，第一个下标是甲的出拳，第二个是乙的出拳
  4. 数组 score 初始化时每一行用 {} 括起来，避免负号引起的编译错误
