前缀和与差分 —— 从概念到公式

发表于 2026-07-20 分类于 算法
本文字数： 5k 阅读时长 ≈ 5 分钟
前缀和与差分是算法竞赛和面试中非常基础的预处理工具。本文用图解法讲清一维/二维前缀和与差分的核心公式，不涉及具体代码，为后面的具体题目做铺垫。

一、为什么需要前缀和？

先想一个场景：你有一个长度为 n 的数组，现在要反复问你「第 3 个数到第 7 个数的和是多少」。

最直接的做法：每次从 3 加到 7。如果数组很大（比如 10 万个数），问你 10 万次，那总共要算 100 亿次加法——太慢了。
如果换一种思路，我能不能提前把「前 i 个数的和」全部算好存起来，这样每次询问直接用减法得到结果？

这就是前缀和的核心思想——预处理，换时间。

三、二维前缀和

3.1 问题升级

如果你面对的不是一维数组，而是一个二维矩阵，想快速求出任意一个子矩形内所有数的和，怎么办？

比如：求从 [2][2] 到 [3][4] 这个 2×3 矩形里所有数的和。

类比一维的思路：二维前缀和就是提前算好从 [1][1] 到 [i][j] 这个矩形内所有数的总和。

3.2 核心公式

定义 s[i][j] = 从左上角 (1,1) 到右下角 (i,j) 的矩形区域内所有元素的和。

3.3 图解：容斥原理

公式看着复杂，其实核心就是容斥原理——两个有重叠的区域合并时，加完两边，减掉重叠。

把 s[i][j] 拆成三块来看：

  • A 区域 = s[i-1][j]：第 1i-1 行、第 1j 列（不含第 i 行）
  • B 区域 = s[i][j-1]：第 1i 行、第 1j-1 列（不含第 j 列）
  • 重叠部分 = s[i-1][j-1]：A 和 B 都覆盖的左上角

四、差分

4.1 差分是前缀和的逆运算

先回顾前缀和的定义：s[i] = a[1] + ... + a[i]。

那反过来，已知 s 怎么求 a？

这就是差分。 对前缀和数组求差分，就得到原数组；对差分数组求前缀和，也得到原数组——两者互逆。

那差分有什么用？看一个场景：

如果要反复给数组的某个区间统一加上一个数，最后再看结果，能不能快速做？

暴力做法是每次从 l 到 r 逐个加，太慢。差分能做到时间复杂度是O(1)， 一次标记，最后统一还原。

七、总结对照

公式速查表

场景 公式 用途
一维前缀和 s[i] = s[i-1] + a[i] 预处理
一维区间和 sum[l,r] = s[r] - s[l-1] 快速查询
二维前缀和 s[i][j] = s[i-1][j] + s[i][j-1] - s[i-1][j-1] + a[i][j] 预处理
二维子矩阵和 sum = s[x2][y2] - s[x1-1][y2] - s[x2][y1-1] + s[x1-1][y1-1] 快速查询
一维差分 b[l] += c, b[r+1] -= c 区间加减
二维差分 b[x1][y1] += c, b[x1][y2+1] -= c, b[x2+1][y1] -= c, b[x2+1][y2+1] += c 子矩阵加减

使用场景

算法 什么时候用 耗时
前缀和 多次查询区间和/子矩阵和 预处理 O(n)，每次查询 O(1)
差分 多次修改区间/子矩阵的值，最后一次性查看结果 每次修改 O(1)，最后恢复 O(n)
前缀和 + 差分 多次修改 + 多次查询 差分做修改，前缀和做查询，中间通过「求前缀和」衔接

关键注意点

  1. 数组下标从 1 开始，避免出现 l-1 = 0 时越界
  2. 设 s[0] = 0（一维）或 s[0][*] = s[*][0] = 0（二维），统一边界条件
  3. 先修改，后查询——差分所有的修改操作做完后，最后求一遍前缀和才能得到最终结果
  4. 二维的「面积加减法」理解了，前缀和和差分的四个公式其实是一个思路，不需要死记硬背

前缀和是「累加」，差分是「标记」——这对互逆的算法体现了预处理和批量操作的核心思想，是算法竞赛和面试中最高频的基础技巧之一。
